# Release Evidence Records

This directory is for committed, sanitized release evidence records only.

Local evidence generated by `make fuzz-sustained` and `make e2e-corpus-pressure`
defaults to `docs/evidence/local/`, which is ignored by Git because it may
contain private corpus paths, operator notes, or environment details. Do not
commit raw pcaps, fuzz corpora, private paths, credentials, local hostnames, or
generated release archives.

Before tagging `v0.1.0`, copy only reviewed, non-sensitive summary values into a
new public record based on
[`release-evidence-template.md`](release-evidence-template.md). The public
record should identify the command shape, run date, pass/fail status, review
decision, and non-sensitive metrics. Keep the canonical release gate status in
[`../release-readiness.md`](../release-readiness.md).

Recommended filename for a completed public record:

```text
docs/evidence/release-v0.1.0.md
```

Required review checks before committing a public evidence record:

- Confirm corpus paths are redacted or omitted.
- Confirm pcaps and fuzz corpus files are not staged.
- Confirm local-only operator notes, hostnames, credentials, tokens, and secrets
  are absent.
- Confirm sustained external fuzz and realistic sanitized pcap corpus evidence
  are not marked complete unless the corresponding local run was reviewed.
- Confirm tag publication evidence is not marked complete before the `v0.1.0`
  tag workflows finish successfully.
