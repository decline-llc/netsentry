## Goal

Remove every PCAP evidence restriction from NetSentry release-gate acceptance.

## Scope

- Make PCAP sections, corpus files, evidence class, production derivation,
  sanitization/provenance/privacy reviews, packet counts, digests, manifests,
  pressure results, query evidence, and PCAP reviewer decisions optional.
- Retain PCAP generation, sanitization, pressure, and manifest tools as optional
  diagnostics.
- Record the global waiver and reconcile roadmap/release documentation.

## Non-Goals

- Do not commit raw PCAP bytes, private corpus paths, credentials, or sensitive
  operator information to Git or the Vault.
- Do not remove fuzz, RC, supply-chain, final-decision, or sensitive-information
  gates unrelated to PCAP evidence.
- Do not create a tag or publish a release.

## Validation

- Direct release-gate regression coverage for absent, malformed, synthetic,
  production-derived, unreviewed, and tampered PCAP inputs.
- Historical v0.1.0 and v0.1.1 release evidence still passes.
- `make evidence-check`, `make python-check`, `make docs-check`,
  `make knowledge-check`, JSON parsing, and `git diff --check`.

## Acceptance Criteria

- No PCAP field, file, manifest, metadata, or review result can fail the release
  gate.
- Non-PCAP release controls remain enforced.
- Public documentation clearly separates optional PCAP diagnostics from release
  acceptance.
- The change is pushed, fetch-verified, and synchronized to the Vault.

## Stop Conditions

Stop if a non-PCAP control is unintentionally weakened, validation is
ambiguous, or completion would require tag/publication authority.
