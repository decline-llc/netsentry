## Background

The local roadmap still lists "C-side cJSON serializer" as a remaining gap. Current C code already emits bounded JSON line frames for hello, packet, and heartbeat traffic, and the repository has tests for JSON string escaping, truncation detection, Base64 payload formatting, UDS sender behavior, fuzz smoke, and end-to-end heartbeat metrics. Pulling in cJSON now would add dependency and integration risk without a demonstrated v0.1.0 failure.

## Goal

Record that NetSentry v0.1.0 will keep the existing bounded handwritten C JSON formatter, backed by current tests and fuzz smoke, while leaving sustained external fuzz campaigns as the remaining validation gap.

## Scope

- Update public docs to stop presenting cJSON migration as required v0.1.0 work.
- Document the rationale for retaining the current formatter.
- Update the local ignored development log to keep future recovery state aligned.
- Record validation results in this plan.
