## Background

The persisted v0.1.0 release task still has three pre-tag blockers: sustained external C fuzz evidence, realistic sanitized pcap corpus pressure evidence, and tag-driven publication verification. The repository has evidence-producing Makefile entrypoints and a public sanitized evidence template, but the actual external fuzz corpus and sanitized pcap corpus are intentionally local-only inputs and must not be committed.

## Goal

Verify the local preconditions for collecting the remaining release evidence, record the current blocker accurately, and keep the release task state from implying that external evidence has been collected when no local external corpus inputs are available in the workspace.

## Scope

- Confirm the repository, Git branch, and approved remote state.
- Check whether workspace-visible `.pcap` or `.pcapng` corpus inputs exist.
- Record the missing operator-supplied corpus inputs in the persisted task state.
- Preserve all remaining release blockers until real evidence is collected and reviewed.

## Non-Goals

- Do not create or push the `v0.1.0` tag.
- Do not change runtime code, APIs, configs, CI workflows, dependencies, or release artifacts.
- Do not commit local evidence under `docs/evidence/local/`.
