## Background

The project governance prompt references a coverage quality gate when a project exposes one. NetSentry currently has `make test`, `make lint`, `make e2e-smoke`, and related release gates, but no native `make test-coverage` target. Adding a Makefile-native coverage target improves quality-gate consistency without changing runtime behavior.

## Goal

Add a native `make test-coverage` target that runs existing C tests and Go coverage with the repository's writable Go cache defaults, then document the target and validate it through the existing quality checks.

## Scope

- Add a root Makefile target named `test-coverage`.
- Use the existing C test target for C validation.
- Generate Go coverage with `go test -coverprofile` and print the total coverage summary.
- Document the target in README, development guide, and changelog.
- Update local task history.

## Non-Goals

- No coverage threshold enforcement in this task.
- No new external coverage tooling.
- No C gcov/lcov integration.
- No core protocol, parser, UDS, SQLite, or API response behavior changes.
- No Docker execution.

## Validation Plan

Run `make test-coverage` first because it is the new target. Then run `make lint`, `go mod verify`, `git diff --check`, and `make e2e-smoke`. If socket operations fail in the sandbox, rerun the same Makefile target with elevated permissions.

## Risk Points

- C UDS tests and e2e smoke need Unix socket operations that may fail inside the sandbox.
- Go coverage with `-coverprofile` and `-race` together is slower; this target intentionally does not use `-race` because `make test` already owns race validation.
- Some Go packages have no test files; the target should still succeed and report total coverage.

## Rollback Plan

Revert the Makefile target and documentation additions. No persistent runtime state or schema changes are involved.

## Acceptance Criteria

- `make test-coverage` exists and prints a Go total coverage line.
- Existing C tests still run as part of the target.
- Documentation lists the target accurately.
- Validation commands pass or have documented sandbox-specific escalation.
- No Docker command is run.

## Execution Results

- `git diff --check` passed.
- Secret scan over changed files found only the documented `${NETSENTRY_API_TOKEN:}` placeholder in `docs/development.md`; no real credentials were found.
- Initial sandboxed `make test-coverage` failed in the C UDS sender test at Unix socket `bind` with `Operation not permitted`, matching the known sandbox socket limitation.
- Elevated `make test-coverage` passed. It ran C parser/UDS tests, generated `/tmp/netsentry-coverage.out`, and reported Go total coverage of 74.2%.
- `make lint` passed. `staticcheck` was not installed and was skipped by the existing Makefile target.
- `go mod verify` passed with all modules verified.
- Initial sandboxed `make e2e-smoke` failed during Python socket creation with `PermissionError: [Errno 1] Operation not permitted`, matching the known sandbox socket limitation.
- Elevated `make e2e-smoke` passed with 6 packets processed, 5 alerts generated, and 8 rules loaded.
- No Docker commands were executed.
