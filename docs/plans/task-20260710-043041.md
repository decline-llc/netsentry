## Background

The v0.1.0 release task has no operator-supplied external fuzz corpus or sanitized
realistic pcap corpus in the workspace. The repository can still produce useful
synthetic extended validation evidence, but that evidence must remain clearly
separate from the external evidence required by the release gate.

The configured `origin` remote also had an SSH push URL while the project
governance requires the approved HTTPS URL for fetch and push.

## Goal

Set both `origin` URLs to `https://github.com/decline-llc/netsentry`, run the
10,000-iteration synthetic repeat-pcap pressure check and 1,000,000-iteration
deterministic ASan parser fuzz evidence check, then update release-readiness
documentation with the results while preserving the external evidence blockers.

## Scope

- Git remote URL configuration.
- Local synthetic pressure and fuzz evidence under ignored `docs/evidence/local/`.
- `docs/release-readiness.md` and this task plan.

## Non-Goals

- Do not create or push the `v0.1.0` tag.
- Do not change runtime code, APIs, configs, workflows, or dependencies.
- Do not commit ignored local evidence or generated release artifacts.
