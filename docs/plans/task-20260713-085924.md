# Task Plan: reject unknown YAML configuration fields

## Metadata

- Timestamp: 2026-07-13T08:59:24Z
- Branch: main
- Risk Level: Medium

## Goal

Make the Go configuration loader fail closed when a YAML document contains an unknown top-level or nested field, preventing misspelled settings from silently falling back to defaults.

## Scope

- Replace the permissive YAML unmarshal path with the existing library's strict known-field decoder.
- Reject multiple YAML documents to keep the file contract unambiguous.
- Add focused tests for unknown top-level and nested configuration fields.
- Document the startup compatibility impact.

## Non-Goals

- Do not wire, remove, or redesign reserved configuration fields.
- Do not change defaults, environment expansion syntax, or runtime configuration values.
- Do not modify the C capture configuration contract.

## Affected Files

- `engine/internal/config/config.go`
- `engine/internal/config/config_test.go`
- `docs/development.md`
- This plan and its task state

## Validation and Stop Conditions

- Run focused Go config tests, `make config-check`, `make docs-check`, `make deps-check`, and `git diff --check`.
- Review changed text for credentials and sensitive absolute paths.
- Stop if strict decoding breaks the repository configuration or requires a configuration-schema decision beyond unknown-field handling.

## Risk and Rollback

Existing configurations with misspelled or unsupported keys will fail at startup; this is intentional fail-closed behavior. Revert this single commit if the decoder creates an unintended incompatibility.
