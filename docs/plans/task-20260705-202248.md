## Validation Plan

The primary validator is full `make rc-check`, with Docker enabled. This should include shell syntax checks, `make test`, `make test-coverage`, `make fuzz-parser`, `make e2e-smoke`, `make dist`, release archive checksum/content smoke, `make docker-build`, Docker image content smoke, and Docker runtime `/api/health` smoke.

## Risk Points

- Docker build may fail due to registry/network instability.
- Unix socket tests require permissions outside the sandbox.
- Docker commands must use sudo.

## Rollback Plan

If validation fails due to external network or Docker daemon availability, record the blocker and do not change source code. If a repository bug is found, create a focused follow-up task and fix only that issue.

- No real secrets are introduced.

## Execution Results

- First full `DOCKER="sudo docker" make rc-check` attempt reached Docker build but failed because non-interactive sudo could not read a password for `sudo docker build`.
- Second full `DOCKER="sudo -S docker" make rc-check` attempt reached Docker image smoke but failed because stdin supplied the sudo password only to the first Docker sudo invocation.
- Third full attempt used a temporary `/tmp` `SUDO_ASKPASS` helper with `DOCKER="sudo -A docker"` so each Docker invocation still executed through sudo. The helper was deleted after the command, and no password was written to the repository.
- Full rc-check passed. Coverage included shell syntax checks, `make test`, `make test-coverage` with Go total coverage 74.2%, `make fuzz-parser`, `make e2e-smoke`, `make dist`, release archive checksum/content smoke, sudo Docker build, sudo Docker image content smoke, and sudo Docker runtime `/api/health` smoke.
- `make lint` passed. `staticcheck` was not installed and was skipped by the existing Makefile target.
- `go mod verify` passed with all modules verified.
- `git diff --check` passed.
- Secret scan over this plan found no matches.
- No source code changes were required.
