## Validation Plan

Primary validation is full `make rc-check` with Docker enabled through sudo. This should include shell syntax, docs, Python, config, dependency, C/Go tests, Go coverage snapshot, deterministic C parser fuzz smoke, e2e smoke, dist archive checksum/content smoke, release notes smoke, Docker build, Docker image content smoke, and Docker runtime `/api/health` smoke.

## Risk Points

- Docker daemon or network access may fail outside repository control.
- Non-interactive sudo can fail if credentials are not primed or cached.
- The RC command may regenerate ignored build artifacts under `bin/` and `dist/`.
- Do not write the sudo password to repository files or committed logs.

## Rollback Plan

If validation produces no source changes, rollback is not needed. If generated tracked files appear unexpectedly, revert only those unrelated generated changes after inspection. If the plan needs removal before commit, delete this plan.

## Validation Result

- Initial elevated `DOCKER="sudo docker" make rc-check` attempt passed all non-Docker gates, then failed at `sudo docker build` because sudo could not read a password without a terminal. This was an execution-mode issue, not a repository build failure.
- Re-ran in a temporary TTY shell, primed sudo with `sudo -S -v`, and executed `DOCKER="sudo docker" make rc-check`; the full RC bundle passed.
- Successful full RC coverage included:
  - shell syntax checks
  - `make docs-check`
  - `make python-check`
  - `make config-check`
  - `make deps-check`
  - `make test`
  - `make test-coverage` with total Go coverage 74.2%
  - `make fuzz-parser` with 5000 deterministic ASan fuzz iterations
  - `make e2e-smoke` with 6 packets processed, 5 alerts generated, and 8 rules loaded
  - `make dist VERSION=0.1.0-dev`
  - release archive checksum/content smoke
  - release notes smoke
  - `sudo docker build -t netsentry:0.1.0-dev .`
  - sudo Docker image content smoke
  - sudo Docker runtime `/api/health` smoke
- Docker build emitted a warning that current commit information was not captured by BuildKit; the image build, content smoke, and runtime health smoke still passed.

## Security Check

The sudo password was provided through stdin in a temporary TTY session and was not written to repository files. No key, token, private key, sudo password, or production secret was added to tracked files.
