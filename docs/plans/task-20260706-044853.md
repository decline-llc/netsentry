## Background

Recent documentation consistency work used one-off `rg` scans to catch stale public documentation phrases. That check is useful enough to preserve as a native quality gate so release-candidate validation can fail early when retired wording returns.

## Goal

Add a repository-native `make docs-check` target backed by a checked-in shell script and include it in `make rc-check`.

## Scope

- Add `scripts/docs_check.sh`.
- Add `docs-check` to the root Makefile.
- Call `make docs-check` from `scripts/rc_check.sh`.
- Update README, development guide, and changelog.
- Record validation in this plan.

## Non-Goals

- No runtime code changes.
- No Docker behavior changes.
- No markdown linter dependency or external tooling.
- No broad style enforcement beyond targeted stale wording checks.
