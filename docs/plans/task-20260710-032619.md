## Background

The remaining v0.1.0 release blockers require sustained external C fuzz evidence and realistic sanitized pcap corpus evidence. The project already generates ignored local JSON and Markdown summaries under `docs/evidence/local/`, but there is no committed public template that defines the safe, reviewable record expected before tagging.

## Goal

Add a public release evidence record template and evidence directory guidance so operators can convert private local evidence into a sanitized, reviewable release record without committing corpus paths, raw pcaps, credentials, or local operator notes.

## Scope

- Add committed `docs/evidence/` guidance.
- Add a release evidence template for sustained fuzz, realistic corpus pressure, local RC validation, and tag publication verification.
- Link the evidence guidance from release readiness, README, and development docs.

## Non-Goals

- Do not create or publish v0.1.0 release evidence.
- Do not claim external fuzz or realistic pcap corpus blockers are complete.
- Do not change evidence scripts, CI workflows, runtime code, APIs, configuration, dependencies, or release tags.
- Do not commit local evidence generated under `docs/evidence/local/`.
