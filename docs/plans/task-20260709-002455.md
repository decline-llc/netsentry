## Background

The v0.1.0 release gate still requires sustained external fuzz evidence and realistic sanitized pcap corpus evidence. The current evidence scripts write local-only summaries under `docs/evidence/local/`, but the JSON and Markdown summaries include absolute corpus paths by default. That is acceptable for ignored local evidence, yet it raises the chance of accidentally publishing private corpus locations when operators extract public summaries.

## Goal

Change the local evidence scripts so generated summaries redact absolute corpus paths by default, while still allowing an explicit local opt-in when full paths are needed for private debugging.

## Scope

- Redact `FUZZ_CORPUS` and `PCAP_CORPUS` absolute paths in generated JSON and Markdown summaries by default.
- Redact per-pcap absolute paths in corpus pressure JSON by default.
- Add opt-in environment variables for local full-path evidence.
- Update public docs to describe the safe default and opt-in behavior.

## Non-Goals

- Do not create or publish release evidence for external corpora.
- Do not alter packet parsing, UDS transport, SQLite storage, API behavior, rule semantics, or release workflows.
- Do not change dependencies.
- Do not commit generated evidence files or raw pcaps.
