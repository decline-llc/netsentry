## Background

NetSentry now has a native `make test-coverage` target that runs C tests and prints a Go coverage summary. The release-candidate bundle in `scripts/rc_check.sh` still runs `make test`, fuzz smoke, e2e smoke, dist, and Docker checks, but it does not yet exercise the coverage snapshot.

## Goal

Add `make test-coverage` to the release-candidate verification bundle and update documentation so maintainers know rc-check includes a coverage snapshot.

## Scope

- Insert `make test-coverage` into `scripts/rc_check.sh`.
- Update README and development guide rc-check descriptions.
- Update CHANGELOG.
- Record validation results in this plan and the local master plan.

## Validation Plan

Use `SKIP_DOCKER=1 make rc-check` to validate the changed rc-check sequence without running Docker. This still exercises shell syntax, `make test`, the new `make test-coverage` step, fuzz smoke, e2e smoke, dist, and archive checks. If Unix socket operations fail in the sandbox, rerun the same target with elevated permissions.

## Risk Points

- `make test` and `make test-coverage` both run C UDS tests that may require elevated socket permissions.
- Full Docker rc-check is not required for this script ordering change, but future full RC runs must still use `DOCKER="sudo docker"`.

## Rollback Plan

Remove the `make test-coverage` rc-check step and revert documentation edits. No runtime state or schema changes are involved.
