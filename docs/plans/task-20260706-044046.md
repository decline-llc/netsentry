## Background

Recent W11/W12 work established `make rc-check` as the release-candidate validation bundle. Dependency integrity is currently verified manually with `go mod verify` in task logs, but it is not exposed as a native Makefile target or included in `scripts/rc_check.sh`.

## Goal

Add a native dependency verification gate and include it in the release-candidate path so dependency integrity is checked every time `make rc-check` runs.

## Scope

- Add a root Makefile target for dependency verification.
- Call the target from `scripts/rc_check.sh`.
- Update public docs that describe release-candidate checks.
- Record this plan and validation outcome.

## Non-Goals

- No Go, C, Dockerfile, or runtime behavior changes.
- No dependency additions, removals, or version changes.
- No new external scanners or network-only tooling.

## Risk Points

- `go mod verify` may need module cache access; use the same Makefile `GOCACHE` and `GOPROXY` defaults as existing Go targets.
- Sandboxed Unix socket restrictions may affect `make rc-check` via C/UDS or e2e tests; if that occurs, rerun with the required elevated execution path and record it.

## Rollback Plan

Revert the Makefile target, `rc_check.sh` step, and documentation updates. No generated runtime data or dependency files should need rollback.
