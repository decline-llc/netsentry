## Background

NetSentry now exports process-lifetime metrics gauges for uptime, packets processed per second, and alerts generated per second. The end-to-end smoke test already validates the pcap -> C capture -> UDS -> Go engine -> SQLite -> API path and asserts core metrics such as capture heartbeat gauges, rule match latency histogram observations, alert write latency histogram observations, and queue high-water presence. It does not yet assert the new process rate gauges on the real smoke path.

## Goal

Update `make e2e-smoke` so it fails if `/api/metrics` does not expose `netsentry_process_uptime_seconds`, `netsentry_packets_processed_per_second`, and `netsentry_alerts_generated_per_second` with sensible values after the synthetic pcap run.

## Risk Points

- The checks must remain tolerant of exact rate values changing across machines.
- Sandbox socket restrictions may require elevated reruns for `make e2e-smoke`.
