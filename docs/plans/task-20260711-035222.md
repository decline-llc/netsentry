## Background

NetSentry already has `make rc-check`, local fuzz/pcap evidence producers, and a public release evidence template. The tag-driven GitHub Release and GHCR workflows currently rerun `rc-check` but do not require a reviewed public evidence record. The remaining release blockers can therefore be made enforceable without changing the pcap parser, UDS protocol, SQLite schema, or API contracts.

## Goal

Add a native `make release-gate` that fails closed unless the reviewed `docs/evidence/release-v0.1.0.md` record contains the required external fuzz, realistic sanitized pcap, query, and sensitive-information approvals. Invoke it before artifact build and image publication in both tag workflows.

## Scope

- Add a standard-library-only release evidence validator.
- Add the `release-gate` Makefile target.
- Call the gate from GitHub Release and GHCR workflows before publishing.
- Document the command and exact acceptance fields.
- Test failure and passing paths in the designated TEST workspace.

## Non-Goals

- No runtime, pcap parser, UDS, SQLite, API, configuration, or dependency changes.
- No automatic creation of evidence or release approval.
- No raw corpus, local evidence, private path, credential, or `v0.1.0` tag.
- No replacement of `rc-check`; the new gate is a release-evidence prerequisite.

## Validation Plan

The gate must fail when the evidence file is missing or contains pending/failed fields, and pass only for a temporary test record with all required fields explicitly approved. The repository must remain free of generated evidence, private paths, dependencies, tags, and core-module changes.

## Risk Points

- A permissive parser could allow a release with incomplete evidence.
- A test fixture could accidentally be committed as real release evidence.
- Workflow ordering could allow artifact publication before the gate.

## Rollback Plan

Revert the validator, Makefile, workflow, and documentation changes as one topic. Do not alter release tags or generated evidence.

## Acceptance Criteria

- Missing evidence fails with a non-zero exit code.
- Evidence with pending/failed review fields fails.
- Explicitly approved evidence with at least 1,000,000 fuzz iterations, zero crashes, passed pcap/query evidence, and all sensitive-information checks set to no passes.
- Both publication workflows call `make release-gate` before publishing.
- All applicable quality gates pass and no release tag is created.
