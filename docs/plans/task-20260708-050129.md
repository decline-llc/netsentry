## Background

The repository already has local release-candidate validation, local release archive packaging, and a GHCR image publication workflow. Public release-readiness docs still list one final blocker: a version tag must drive the named GitHub Release and named registry image publication. The registry image side exists in `.github/workflows/docker-publish.yml`, but the repository does not yet create a GitHub Release with the packaged tarball and checksum.

## Goal

Add a standard tag-driven GitHub Actions workflow that reruns the release-candidate bundle, builds the release archive, and publishes a GitHub Release containing the generated tarball and checksum. Expose the flow in local docs and Makefile help without changing runtime behavior.

## Scope

- Add a GitHub Actions workflow for version-tag release publication.
- Add a lightweight local Makefile helper that validates release tag format before invoking existing artifact generation.
- Document the release flow in public docs.

## Non-Goals

- Do not create or push a real version tag in this task.
- Do not publish a real GitHub Release or GHCR image from this environment.
- Do not change C capture, Go engine, API behavior, SQLite schema, or rule semantics.
- Do not add third-party runtime dependencies.
