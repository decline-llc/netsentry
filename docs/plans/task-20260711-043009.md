## Background

The available corpus is deterministic synthetic data and cannot satisfy the internal production-derived traffic gate without a separately approved exception. The project generator currently creates only six files, with identical outputs across repeated directories. The next safe step is to produce larger, deterministic, non-sensitive synthetic pressure input and record it only as auxiliary evidence.

## Goal

Extend the existing standard-library generator to create a caller-selected number of differentiated synthetic corpus sets, verify reproducibility and unique hashes, and run the existing corpus-pressure harness in the TEST project copy. Keep the release gate and tag publication blocked pending explicit administrator approval.

## Scope

- Extend `scripts/gen_sanitized_corpus.py` with deterministic batch generation.
- Add a native Makefile parameter for batch size.
- Update generator documentation and evidence-boundary wording.
- Generate a temporary 100-set corpus outside Git.
- Run `make e2e-corpus-pressure` against the temporary corpus in the designated TEST workspace.

## Non-Goals

- No claim that synthetic evidence is production or external evidence.
- No release-gate bypass or permanent exception record without explicit approval.
- No raw pcaps, local evidence, or private paths committed.
- No parser, UDS, SQLite, API, or release-tag changes.

## Validation Plan

For 100 sets, the generator must emit 600 files (300 pcap and 300 pcapng), all unique by SHA-256 within the batch, with repeat runs byte-identical. The pressure harness must finish with zero UDS write errors and expected process-rate metrics. The resulting evidence must remain local-only and synthetic.

## Risk Points

- Large file count may make the pressure harness slow; stop without weakening checks if runtime is excessive.
- Synthetic results could be misread as production evidence.
- Generated corpus or absolute paths could be staged accidentally.

## Rollback Plan

Revert the generator, Makefile, docs, and plan changes as one topic. Delete only temporary corpus/evidence directories; do not touch runtime data.
