## Background

The v0.1.0 release readiness checklist still requires sustained external C fuzz evidence. The repository already has deterministic ASan parser fuzz smoke and a longer local target, but there is no standard evidence-producing command for release-candidate fuzz runs or optional external corpus files.

## Goal

Add a native Makefile entrypoint and shell script that run the existing ASan C parser fuzz harness in sustained mode, optionally replay external corpus files, and write local JSON/Markdown evidence without committing private corpus paths or samples.

## Scope

- Add `scripts/fuzz_sustained.sh`.
- Add root `make fuzz-sustained`.
- Include the new script in shell syntax checks.
- Document usage, release-readiness impact, and evidence output location.

## Non-Goals

- Do not change parser behavior or fuzz semantics.
- Do not add new fuzz engines or external dependencies.
- Do not commit external corpus files.
- Do not claim that release evidence is complete until a real sustained run is recorded.
