## Background

The knowledge vault has accurate core notes but the generated full commit index remains at 180 entries while the repository contains 189 commits. The current local post-push hook writes per-commit notes but does not rebuild the full index, extract the current fuzz/corpus/deferred metadata, or update the MOC with changed documentation entries.

## Goal

Make the SSH post-push hook idempotently synchronize all pushed Git history into the external Obsidian vault: regenerate the full index, create one iteration note per push range with current evidence metadata, and add unique MOC links for changed technical documents.

## Scope

- Replace the local `.git/hooks/post-push` implementation.
- Add a standard-library Python synchronizer embedded in the hook for deterministic Markdown generation.
- Rebuild `04-开发迭代记录/全量提交索引.md` from `git log --all`.
- Generate push-range iteration notes and `_待整理` drafts with approval/fuzz/corpus/deferred metadata.
- Update MOC entries from changed repo paths using a stable mapping.
- Update `docs/knowledge-management.md` with the new behavior and manual command.

## Non-Goals

- No third-party dependency or Obsidian CLI requirement.
- No modification of generated pcap binaries or ignored evidence files.

## Risk Points

- Vault is external to Git and must be treated as generated state.
- Git post-push provides new remote SHA; the hook must derive the old SHA safely from local reflog or explicit range.
- The full index must not expose private paths or credentials.
- Hook failures must be visible but non-blocking after a successful push.

## Rollback Plan
