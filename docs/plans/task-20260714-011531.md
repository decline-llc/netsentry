# Task Plan: remove unconsumed YAML configuration fields

## Metadata

- Timestamp: 2026-07-14T01:15:31Z
- Branch: main
- Risk Level: Medium

## Goal

Make the Go engine YAML configuration contract fail closed: every accepted field has a runtime consumer, while the standalone C capture binary retains its explicit command-line contract.

## Scope

- Remove the unconsumed `capture` YAML section and document the C capture CLI boundary.
- Move UDS socket permissions to the engine section, where the receiver applies them.
- Remove unimplemented CORS and aggregation-limit settings plus unused logging fields.
- Update repository and E2E configuration fixtures.
- Add regression tests that each removed field is rejected by strict decoding.

## Non-Goals

- Do not add a YAML parser or new dependency to the C capture binary.
- Do not implement CORS, aggregation caps, configurable log sinks, or C capture heartbeat/payload settings.
- Do not change defaults or runtime behavior for retained settings.

## Validation and Stop Conditions

- Run focused Go config tests, `make config-check`, `make docs-check`, `make shell-check`, `make deps-check`, `make e2e-smoke`, and `git diff --check`.
- Review changes for credentials and sensitive absolute paths.
- Stop if a removed field has an overlooked runtime consumer or a shared C/Go configuration decision is required.

## Risk and Rollback

Existing configurations that contain removed reserved fields will now fail at startup. Revert this one commit to restore the previous, misleading acceptance behavior.
