# Task Plan: decouple knowledge sync tests from Git hooks

## Metadata

- Timestamp: 2026-07-14T07:45:34Z
- Branch: main
- Risk Level: Medium

## Goal

Make post-push knowledge synchronization directly testable from versioned Python code so CI does not require an untracked `.git/hooks/post-push` file.

## Scope

- Extract a direct sync API from the versioned synchronizer.
- Change tests to call the API directly with temporary repositories and Vaults.
- Replace the local hook implementation with a thin Python-entrypoint wrapper.
- Preserve explicit local hook deployment and exact-range usage.

## Non-Goals

- Do not provision hooks in CI.
- Do not remove the local hook or the local Vault workflow.
- Do not begin the blocked production-traffic evidence task.

## Validation and Stop Conditions

- Run focused synchronizer tests, hook syntax validation, `make python-check`, `make knowledge-check`, `make docs-check`, and `git diff --check`.
- Stop if the direct API changes the expected local exact-range synchronization behavior.
