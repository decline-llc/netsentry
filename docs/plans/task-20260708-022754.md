## Background

W11-W12 release readiness still lists realistic pcap corpus pressure/query evidence as a blocker. The current `make e2e-pressure` target is valuable but uses a synthetic repeated six-packet pcap, so it cannot record evidence from sanitized external corpora.

## Goal

Add a repeatable local entrypoint for running the existing offline pcap pipeline against a user-supplied pcap file or directory of sanitized pcaps, and document how to record the resulting JSON/Markdown evidence without committing the corpus.

## Scope

- Add a shell script that starts the engine once, runs capture for each pcap, waits for processing, queries health/metrics/alerts, and writes evidence artifacts.
- Add a Makefile target for the script.
- Document usage and release-readiness impact.

## Non-Goals

- Do not commit real pcap files.
- Do not change detection behavior, rule semantics, storage schema, or API contracts.
- Do not add new external dependencies.
- Do not include this optional corpus run in `make rc-check`.
