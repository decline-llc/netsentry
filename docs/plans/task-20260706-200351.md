## Background

NetSentry release-candidate validation already includes shell syntax, Python syntax, documentation consistency, dependency verification, tests, coverage, fuzz, e2e, archive, and Docker checks. Repository configuration files are covered indirectly by unit and smoke tests, but there is no native target that explicitly validates the checked-in runtime config, rule samples, and suppressions file.

## Goal

Add a native `make config-check` gate that validates checked-in config artifacts through the existing Go parsing and compilation code paths, then include it in `make rc-check`.

## Scope

- Add a root Makefile target named `config-check`.
- Reuse focused Go tests for repository config, rule files, and suppressions files.
- Add missing suppressions repository-file validation coverage.
- Include `make config-check` in `scripts/rc_check.sh`.
- Update README, development guide, changelog, and this plan.

## Non-Goals

- No runtime behavior changes.
- No API, protocol, SQLite schema, or Docker behavior changes.
- No new dependencies or external validators.
- No broad schema-generation work.
